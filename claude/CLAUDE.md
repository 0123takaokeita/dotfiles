# グローバルユーザー指示


## Git 運用

- **`git rebase` は絶対に実行しない**（`git rebase`、`git pull --rebase` を含む）。ブランチが上流（develop / main 等）から遅れていても、勝手にリベースで追従させない。
  - **理由**: ユーザーが明確かつ強く禁止。リベースは履歴を書き換えるため、共有ブランチや進行中の作業で予期せぬ分岐・復旧作業を招く。過去に勝手にリベースして `reset --hard` で巻き戻す事態になった。
  - **代わりにどうする**: 上流の変更を取り込む必要がある場合は `git merge` を提案する（取り込み自体が必要かを先に確認する）。`git pull` する際も `--rebase` を付けない。「古くなってる」と気づいても、リベースを実行せず、まずユーザーに方針を確認する。
