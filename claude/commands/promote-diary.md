# Promote Diary — Task Diary から CLAUDE.md への昇格

Task Diary に蓄積された学びを分析し、共通パターンを CLAUDE.md に昇格します。

## 手順

### 1. Task Diary を読み込む

現在のプロジェクトの当月 Task Diary を読み込む：
`/Volumes/EXSSD/dev/github.com/0123takaokeita/vault/claude/task-diary/<project-name>-<YYYY-MM>.md`

パターン分析のために過去月のファイル（同プロジェクトの `<project-name>-*.md`）も存在すれば合わせて読み込む。

### 2. パターンを分析する

全エントリを横断して以下を見つける：

- **複数回登場した発見・ルール**（2回以上 → 昇格候補）
- **毎回うまくいった手法**（確定ルール化できるもの）
- **毎回ハマった落とし穴**（避けるべきパターン）

### 3. 昇格候補をユーザーに提示する

昇格候補のリストを提示し、どれを CLAUDE.md に追加するか確認を取る：

```
以下を CLAUDE.md に昇格しますか？

1. [ルール名]: [内容]
2. [ルール名]: [内容]
...

追加したいものの番号を教えてください（複数可）。
```

### 4. 確認後、CLAUDE.md に追記する

ユーザーが承認した項目を、現在のプロジェクトの CLAUDE.md（存在しない場合は `~/.claude/CLAUDE.md`）の適切なセクションに追記する。

追記形式：
```markdown
## [セクション名（既存に合わせる）]

- [ルール内容]（Task Diary より昇格: YYYY-MM-DD）
```

### 5. 昇格済みエントリにマークをつける

Task Diary のエントリで昇格したものに `✓ 昇格済み` のタグを追加する。

### 6. 完了を報告する

何件昇格したかを報告する。
